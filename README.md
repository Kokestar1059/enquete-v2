# ①課題名

鵠沼海岸まちづくりアンケート（AI分類・分析アプリ）

## ②課題内容（どんな作品か）

- 鵠沼海岸についての「頻度・目的・不満」の3問に、AIが会話形式でインタビューするアンケートアプリ。
- 集めた自由回答を AI（Azure OpenAI）が5カテゴリ（インフラ／自然環境／安全／飲食・施設／その他）に自動分類し、CSVに保存する。
- 回答一覧に加え、カテゴリ別の件数を棒グラフで視覚化。「AIで分析する」ボタンで、回答全体の傾向・要点・改善案を AI が文章でまとめる。
- フレームワークなしの素のPHP＋CSV保存。PHPの基礎と Git/GitHub の習得を目的とした学習課題。

## ⑤工夫した点・こだわった点

- **APIキーを絶対にブラウザへ出さない**：JSは自前の `chat.php` / `analyze.php` を呼び、キーはサーバー側のPHPだけで使う「中継方式」にした。
- **AIの“表記ゆれ”をPHP側で吸収**：分類でAIが「インフラ（駐車場…）」のようにカッコ説明ごと返しても、5カテゴリに正しく矯正される処理を入れた。
- **CSVの列ずれ対策**：自由記述に含まれるカンマ・改行を置換し、`explode(",")` 方式でも崩れないようにした。
- **日本語入力(IME)の確定Enter対策**：変換確定のEnterで未確定文字が送信されないよう `isComposing` で制御。
- **コスト/待ち時間への配慮**：AI分析は「ボタンを押したときだけ」実行（ページを開くたびにAPIを叩かない）。
- **集計の視覚化**：カテゴリ別件数をCSSの横棒グラフで表示。

## ⑥難しかった点・次回トライしたいこと（又は機能）

- **PHPからのAPI呼び出し（cURL）** が初挑戦で難しかった。リクエスト本文の組み立て、ヘッダでのキー指定、レスポンスJSONの取り出しを一つずつ理解した。
- **Azure OpenAI → Azure AI Foundry の仕様変更**：キーの取得先やエンドポイント形式、v1 API（`api-version` 不要）の違いを調べて対応した。
- 次回トライしたいこと：ログイン機能、CSVではなくデータベース（MySQL等）での保存、グラフライブラリでのリッチな可視化、回答のやり直し・編集機能。

## ⑦フリー項目（感想、シェアしたいこと等なんでも）

- [感想] 「JSのfetch ≒ サーバー中継」「`$_POST` ≒ リクエスト受け取り」「`fwrite` ≒ insert」「`fgets` ≒ select」と、これまでの経験に対応づけながら進めたことで、素のPHPの流れが腑に落ちた。AIの返事をそのまま信用せずPHP側で検証・矯正する、という考え方が学びになった。
