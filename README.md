# ①課題名

鵠沼海岸まちづくりアンケート（AI分類・分析アプリ）― データベース版（v2）

## ②課題内容（どんな作品か）

- 鵠沼海岸についての「頻度・目的・不満」の3問に、AIが会話形式でインタビューするアンケートアプリ。
- 前課題（CSV保存版）を土台に、**保存先を CSV → MySQL（データベース）へ移行**した続編。
- 集めた自由回答を AI（Azure OpenAI）が5カテゴリ（インフラ／自然環境／安全／飲食・施設／その他）に自動分類してDBに保存する。
- 回答一覧とカテゴリ別件数の棒グラフを表示。「再分析」ボタンで AI が全体の傾向・要点・改善案を文章でまとめ、**結果もDBに保存**する。通常表示は保存済みの分析を読むだけ（毎回APIを呼ばない）。
- フレームワークなしの素のPHP＋PDO（MySQL）。

## ③工夫した点・こだわった点

- **APIキーを絶対にブラウザへ出さない**：JSは自前の `chat.php` / `analyze.php` を呼び、キーはサーバー側のPHPだけで使う「中継方式」にした。
- **CSV → MySQL へ移行**：PDO＋プリペアドステートメントで値を渡し、SQLインジェクションを防止。自由記述のカンマ・改行による列ずれの心配も無くなった。
- **カテゴリの正規化**：カテゴリ名を `categories` マスタに集約し、`responses` からは外部キー（`category_id`）で参照。一覧・集計は JOIN／GROUP BY で取得。
- **AI分析のコスト配慮**：分析結果をDBに保存し、通常は保存済みを表示。「再分析」ボタンを押したときだけ API を呼ぶ。
- **連打対策（レート制限）**：公開エンドポイントを短時間に連打されてもAzure使用量が暴発しないよう、サーバー側で回数制限をかけ超過時は429を返す。

## ④難しかった点・次回トライしたいこと（又は機能）

- **CSVからDBへの移行とテーブル設計**：正規化・外部キー・JOIN を実際に手を動かして理解した。既存データを壊さず文字列カテゴリを `category_id` へ移し替える手順が学びになった。
- 次回トライしたいこと：1回答＝複数カテゴリ（多対多・中間テーブル）、グラフライブラリでのリッチな可視化、回答の編集・やり直し機能。

## ⑤フリー項目（感想、シェアしたいこと等なんでも）

- 「`fwrite` ≒ INSERT」「`fgets` ≒ SELECT」「PDO ≒ Supabaseクライアント」と、これまでの経験に対応づけながら進めたことで、CSVからDBへの置き換えが腑に落ちた。
